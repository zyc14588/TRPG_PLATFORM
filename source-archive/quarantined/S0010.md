# V6 Codex 第四次严格修复报告：输出路径一致性

## 1. 修复目标

本轮修复专门处理上一轮严格复核发现的 P0/P1 问题：

```text
257 个 prompt 的建议 Rust module 与实际建议输出文件不一致。
34 个实际建议输出文件名存在截断尾缀。
部分具体 Rust 输出路径被多个 prompt 复用，且缺少 primary / supplemental 归并规则。
archived module-layout validation 未覆盖 output path consistency。
```

## 2. 修复规则

- `product-code`、`test-harness`、`ops-runbook` 的 Rust 输出文件由 `建议 crate` 与 `建议 Rust module` 唯一推导。
- primary implementation prompt 才允许声明 `crates/<crate>/src/<module_tail>.rs` 与对应 contract test。
- supplemental requirement prompt 不再声明或创建 `.rs` 文件，只输出补充需求追踪文档。
- `docs-governance` 与 `traceability-maintenance` prompt 只维护 Markdown、索引、矩阵、报告或追踪材料。
- 禁止源文档路径截断名、旧 V5 中间文件名、hash 片段进入 Rust 输出路径。
