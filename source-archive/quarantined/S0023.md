## 1. 修复目标

本轮针对上次严格复核发现的剩余问题进行第五次小范围修复：

1. 重写 supplemental prompt 的 Codex 编码 / 测试 / review 提示词，确保 supplemental 不再要求创建或实现 Rust module。
2. 将 source-path-like primary Rust module 改为领域稳定名，并同步更新 `src/<module>.rs` 与测试输出路径。
3. 清理短 hash 残片和版本/strict/source 路径片段，确保它们只留在 Prompt ID、源 SHA 或追踪字段中。
4. 对重复 primary 输出 owner 执行 primary/supplemental 归并，保证同一 concrete Rust 输出路径只有一个 primary owner。
5. 扩展最终 validation，使语义层冲突纳入严格验收。

## 3. 新增施工规则

- `primary-implementation` 是唯一可以声明 Rust src/test 输出文件的角色。
- `supplemental-requirement` 只维护 `docs/codex/90-traceability/supplemental-requirements/<Prompt ID>.md`。
- supplemental 中的代码级要求必须写成“合并到 Primary Prompt”的补充约束。
- primary module 必须是稳定领域名；`docs_*`、`generated_*`、`source_*`、`*_v4`、`*_strict_v4`、截断尾缀和 hash 残片不得进入 module 或文件名。
- 同一 concrete Rust 输出路径只能有一个 primary owner。
