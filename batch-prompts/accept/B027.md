# BATCH-027 手动验收提示词参考

## 用途

本文件供人类程序员复制到 Codex，用于严格验收 `BATCH-027-06-data-eventing — Strict Governance Final` 是否已按设计、测试和 traceability 要求完成。验收阶段不得擅自追加新功能，只能报告 PASS/FAIL；如需修复，必须输出最小修复提示词。

## 必读输入

1. `batches/B027.md`
2. 当前 batch 引用的全部 per-file prompt。
3. `evidence/batches/BATCH-027/plan.md`
4. `evidence/batches/BATCH-027/changed-files.txt`
5. `evidence/batches/BATCH-027/test-output.txt`
6. `evidence/batches/BATCH-027/prompt-traceability.md`
7. 所属阶段的 `ACCEPTANCE_PROMPT.md`、`TEST_PLAN.md` 与 `TEST_DATA.md`。

## 可复制给 Codex 的中文提示词

```text
你是 Codex，正在对 `BATCH-027-06-data-eventing — Strict Governance Final` 执行严格验收。

验收规则：
- 验收时不得实现新功能；除非报告 FAIL 后用户明确要求修复，否则只能检查、复跑测试、整理证据和输出修复建议。
- 重新读取 `batches/B027.md`、其引用的 per-file prompts、`AGENTS.md`、`docs/codex/00-index/CURRENT_NORMALIZED_PROMPT_EXECUTION_MAP.md`、`docs/codex/00-index/CURRENT_SAFE_MODULE_AND_OUTPUT_MAP.md` 与 `docs/codex/00-index/CURRENT_TOKEN_REWRITE_TABLE.md`。
- 检查所有变更命名是否使用 current-safe module/output，不得重新引入历史 V3/V4/V5/V6 当前语义。
- 检查实现是否保持 Authority Contract 不可变、Agent Gateway-only AI access、Tool Permission Gate、Visibility Label Propagation、Fact Provenance、Event Log 与 V1 Acceptance 边界。
- 检查 `evidence/batches/BATCH-027/` 证据，并复跑当前 batch 所属阶段要求的相关测试。

必须检查：
1. batch 中每条 prompt row 都有验收结论。
2. 每个 primary prompt 都有实现证据，以及至少一个目标测试或明确非代码理由。
3. supplemental prompt 没有擅自扩展实现 scope。
4. 没有在 Agent Runtime / Provider Adapter 之外引入直接 LLM/provider 调用路径。
5. 没有正式游戏状态绕过 State Service / Event Log 写入。
6. 没有 keeper_only、private_to_player 或 ai_internal fixture 泄露到 player-visible 输出。
7. 所属阶段相关 cargo、pnpm、docker、fixture 检查已经运行；若不适用，必须写明理由。

输出严格结果：
- 所有检查通过时才能给出 PASS。
- 任一检查失败时必须给出 FAIL，并列出 P0/P1/P2 级别、精确文件、证据和可复制的最小修复提示词。
```

## 预期证据

- `evidence/batches/BATCH-027/acceptance-report.md`
- `evidence/batches/BATCH-027/acceptance-test-output.txt`
- 若 FAIL：`evidence/batches/BATCH-027/repair-prompt.md`

## PASS 标准

必须同时满足：prompt traceability 闭合、目标测试通过、无未授权 scope 扩张、无历史当前命名回归、无顶层设计红线违反。
